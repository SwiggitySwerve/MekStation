## Risky Suppression And Fallback Areas

- `src/simulation/known-limitations.md` says it is not the feature-status source of truth. It still retains broad legacy detector buckets for physical attacks, ammo, heat shutdown, terrain movement, pilot skill checks, critical hits, LOS, and SPAs.
- Catalog validation bypasses known-limitation filtering via `battlemech-combat-validation`; losing that bypass would be risky because generic detector messages can still be filtered.
- Static weapon database, synthetic Medium Laser fallback, broad catalog filtering, variable missile damage strings, and zero-damage special rows are explicitly risky fallback areas. They are covered as validation traps, not as implementation shortcuts.
- Out-of-scope rows must remain separate. The 121 out-of-scope rows should not be counted as BattleMech completion, especially aerospace, vehicle, infantry, battle armor, ProtoMech, swarm/anti-Mek, artillery, torpedo, control-plane rows, experimental/unofficial BattleMech-family ammo scope splits, Claw/Talon producer lifecycle scope, Plasma Cannon non-Mek/terrain/building scope, Oblique Artillery scatter scope, Filtration Implants environment/personnel hazard scope, and MekStation legacy Toughness aliases.
- Helper-only rows are deliberate blockers. Source refs plus helper evidence do not equal integrated runtime/interactive parity.
