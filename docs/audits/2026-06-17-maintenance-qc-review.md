# Maintenance QC Review

Date: 2026-06-17

## Scope

Post-merge maintenance review for MekStation, with organization first:

- Create a durable QC graph for top-level, main-module, and submodule review.
- Run read-only maintenance scanner lanes across bloat, type safety, dead code,
  imports, TODOs, lint/build, coverage gaps, complexity, smells, design
  violations, and duplicates.
- Validate the tactical-map and BattleMech combat claims against current
  code/tests/OpenSpec.
- Apply only narrow safe repairs needed to restore validations.

Open Brain was not used.

## New QC Surfaces

- `docs/qc/mekstation-qc-registry.json`: machine-readable QC registry with 24
  surfaces.
- `docs/qc/mekstation-qc-map.md`: human-readable QC graph and lane guide.
- `scripts/qc/validate-qc-registry.mjs`: structural registry validator.
- `scripts/qc/select-qc-lanes.mjs`: query tool for future on-demand QC lanes.
- `docs/qc/maintenance-baseline.json`: reviewed maintenance scanner baseline
  for regression gating while broad debt lanes are burned down deliberately.
- `scripts/maintenance/scan-maintenance.mjs`: on-demand maintenance scanner.
- `.gitignore`: keeps broad investigation scripts ignored while allowing the
  durable QC scripts above to be tracked.
- `package.json`: adds `qc:validate`, `qc:select`, `maintain:scan`, and
  `maintain:scan:gate`.
- Tactical/combat QC lanes now carry stable `claimIds`, and `qc:select`
  supports `--claim=<id>` for on-demand validation of a specific checklist
  promise.
- The simulation/combat branch now has explicit physical weapon runtime,
  ejection lifecycle, and non-BattleMech scope-matrix QC sublanes so future
  reviews do not have to infer those boundaries from stale audit prose.

Validated examples:

```powershell
npm.cmd run qc:validate
npm.cmd run qc:select -- --risk=rules-parity --status=partial
npm.cmd run qc:select -- --claim=ui.tactical.movement-preview
npm.cmd run qc:select -- --text=tactical
npm.cmd run maintain:scan:gate
```
