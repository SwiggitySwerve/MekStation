# Combat Validation Source-Truth Cross-Check

## Verdict

The three-lane validation direction is still right: catalog contracts, behavior-class tests, and representative runner/interactive scenarios. The correction is that catalog presence is not rule truth. A row is only "integrated" when it has both local implementation evidence and an explicit authority boundary.

## Authority Model

- Rulebook authority: Total Warfare / BattleMech Manual style rules define the tabletop mechanic.
- MegaMek source authority: use local MegaMek as executable reference for exact behavior only after reading the relevant source.
- MekHQ behavior authority: use only for campaign/maintenance behavior, not tabletop combat effects.
- MekStation deviation: UI, networking, event ordering, target filters, objectives, and turn-rotation semantics are product contracts layered over tabletop outcomes.

## Council Cross-Exam Findings

- OMO-native council seats were unavailable in this Codex app runtime, so the review fell back to an engineering-council shape with real read-only subagents for source truth, invariants, and critique.
- The current push/charge displacement implementation is aligned with the verified MegaMek source shape, but the surrounding requirement rows must not read as full physical-combat parity.
- Interactive/event-sourced physical resolution now emits and replays successful push/charge displacement payloads when a grid is supplied, closing the runner-vs-interactive split-brain for that successful-hit shape.
- Push/charge/DFA legality gates are now cataloged against MegaMek source, and the representable push/DFA posture/elevation/facing/unit-type gates plus the BattleMech displacement elevation cap have started moving into shared restriction helpers. MegaMek still checks target/entity type, range, movement state, full displacement consequences, and building/invalid target shapes that MekStation does not yet enforce.
- Charge-miss and valid DFA hit/miss displacement now emit from runner and event-sourced resolution; blocked successful charge displacement is source-backed as a no-displacement/no-destruction/no-charge-PSR branch after charge damage; runner same-phase grid occupancy now refreshes after displacement payloads; DFA impossible-displacement destruction now emits with an explicit destruction cause; and valid missed-DFA resolution now immediately applies fall damage, prone/facing state, `UnitFell`, and fall pilot-damage avoidance instead of queuing the legacy miss PSR.
- MegaMek's physical target probe includes additional physical classes such as brush-off, thrash, lay explosives, trip, grapple, break grapple, jump-jet attacks, ProtoMek physical, and Battle Armor vibro-claw paths. These are now explicitly cataloged as integrated, unsupported BattleMech-applicable, or non-BattleMech scope rather than silently absent.
- MekHQ should remain low priority for combat mechanics. Use it only when a campaign, maintenance, or quirk lifecycle is specifically in scope.

## Gates To Keep

- Do not promote a requirement to integrated if any support reference is helper-only or unsupported.
- Do not use OpenSpec alone as external rule authority; it is the project contract.
- Do not use broad known-limitation filters to certify combat coverage.
- Do not claim non-BattleMech coverage in the BattleMech matrix. Vehicles, aerospace, infantry, battle armor, and ProtoMechs need separate matrices or explicit out-of-scope rows.
- For lifecycle claims, require command/intent/wire coverage, event emission, state mutation, target filtering, and terminal summary evidence.
- Feature support rows that put `source-backed` in their evidence must carry structured `sourceRefs` with pinned rulebook or MegaMek/MekHQ authority links; prose-only authority claims are not enough.
- Helper-only or unsupported combat classes must have a named authority row, an expected simulation scenario, and an explicit reason for exclusion from the BattleMech matrix.
- Physical legality rows must carry commit-pinned MegaMek `sourceRefs`; unsupported rows may stay unsupported, but they cannot stay source-ambiguous.
