### Classification corrections discovered during execution

1. **RC9's real mechanism was NOT a hydration race.** `useServiceWorker`
   reloaded the window on EVERY `controllerchange` — including the service
   worker's FIRST install (`clients.claim()`, service-worker.js:39). Every
   Playwright test runs in a fresh browser context, so a spurious full
   reload landed mid-test, silently unmounting the menu / swallowing
   clicks (probe: trusted click no-op while `el.click()` worked; page
   instrumentation wiped mid-test). **Production fix:** reload only when an
   EXISTING controller is replaced (canonical PWA pattern), with red-first
   Jest tests (useServiceWorker.test.ts). This also removes a real
   first-visit reload every fresh visitor experienced.
2. **/units never hydrates — client bundle pulls better-sqlite3** (its
   `lib/methods/backup.js` runs `promisify(fs.access)` at module eval
   against the `emptyModule` fs stub → `TypeError: The "original" argument
must be of type Function`; React unwinds before `_app`'s effects).
   Verified pre-existing on the clean baseline via stash-probe. Follow-up
   row **T2-F2**. Until fixed: mobile-navigation :42 asserts header
   presence only (SSR-rendered; no hydration wait) and the :354 nav flow
   routes via /compendium → /gameplay/pilots → /customizer.
3. **RC4 masked a second layer of drift**: beyond the non-reactive list
   read, the campaign detail tests pinned the unmounted CampaignOverviewTab
   surface (tabs/resources-card/campaign-status/delete dialog). Re-pinned to
   CampaignDashboardPage; the UI delete flow no longer exists anywhere →
   **T2-F3**.
4. **RC17 had a third defect** behind the two classified ones: the seeder
   helper serialized its worker callback via `toString()`, so closure
   variables never reached the page (`ReferenceError: offers is not
defined`). Rebuilt as payload-passing `applyCampaignUpdate`.
5. **`gotoWithRetry` added** (e2e/helpers/navigation.ts): Next dev
   transiently aborts in-flight navigations while on-demand-compiling cold
   routes (`net::ERR_ABORTED`) — bounded 3-attempt retry, mirrors the
   pre-existing quick-play pattern.
