# Spacing & Layout

MekStation uses Tailwind's spacing scale with additional custom values for touch targets.

## Spacing Scale

The default Tailwind spacing scale (in rem/pixels):

| Class | Value | Pixels |
|-------|-------|--------|
| `p-0` | 0 | 0px |
| `p-0.5` | 0.125rem | 2px |
| `p-1` | 0.25rem | 4px |
| `p-1.5` | 0.375rem | 6px |
| `p-2` | 0.5rem | 8px |
| `p-2.5` | 0.625rem | 10px |
| `p-3` | 0.75rem | 12px |
| `p-3.5` | 0.875rem | 14px |
| `p-4` | 1rem | 16px |
| `p-5` | 1.25rem | 20px |
| `p-6` | 1.5rem | 24px |
| `p-7` | 1.75rem | 28px |
| `p-8` | 2rem | 32px |
| `p-10` | 2.5rem | 40px |
| `p-12` | 3rem | 48px |
| `p-16` | 4rem | 64px |

## Touch Targets

Custom spacing for mobile touch targets (44px minimum per iOS/Android guidelines):

```css

min-w-touch: 44px;
min-h-touch: 44px;
```

## Margin & Padding Best Practices

1. **Use consistent increments** - Stick to 4px, 8px, 16px, 24px, 32px
2. **Mobile-first** - Start with smaller spacing, increase for larger screens
3. **Touch targets** - Ensure 44px minimum for interactive elements on mobile
4. **Breathing room** - Cards and sections need adequate internal padding
5. **Visual hierarchy** - Use larger gaps between sections, smaller within
