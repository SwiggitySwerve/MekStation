## Lessons captured

- **The hydration mismatch in PT-102 had survived all prior CI runs.** Jest doesn't exercise SSR + hydration; Storybook doesn't either; existing e2e tests didn't have a campaign in localStorage when first hitting the list page. The Phase-3 spec's "load page with persisted state and assert no console errors" pattern is the smallest possible asser­tion that catches this class of bug. Worth replicating across the rest of the page tree.
- **The state-cycle detector is 96% false-positive on real swarm runs.** Worth a Wave-6 OpenSpec change. The 125 hits / 130 runs ratio is unambiguous evidence.
- **Vault auth is the wall** for both MP and co-op scripted E2E. Either route surfaces past the wall (low-cost — UI route work) or test-mode auth bypass becomes a thing. Both are gap candidates.
- **One PR per defect + scripted spec per phase** kept review surface tight and atomic. Each PR shipped a defect fix + the spec that caught it. The full session is reconstructable from PR history.
- **Background CI + scheduled wakeups** kept the session productive across compile delays. Roughly 3× faster than blocking-wait would have been.
