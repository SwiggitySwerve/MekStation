## Lessons captured in MEMORY

- Phase 0 of any future playtest will surface ~10-30% of e2e specs as broken due to UI drift accumulated since they were authored. Don't conflate with Wave-N defects.
- "Defects only; log gaps" needs a third bucket: **test-staleness** (spec code refers to removed methods or renamed labels). Treat as P1-P3 maintenance, not as a regression.
- The custom `server.js` HMR-upgrade gap (PT-005) was a latent dev-experience bug masked because the e2e test ran with `npm run dev`. Production builds bypass the issue entirely. Worth re-checking on future Wave-N's custom-server changes.
