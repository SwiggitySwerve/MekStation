## Driver bug surfaced + fixed this session

`scripts/swarm-configs/smoke/run-matrix.js` initially reported all 15 configs as FAILED. Root cause: `execFileSync` throws on non-zero exit code, and `run-simulation.ts` exits 1 whenever any run records violations (normal for the smoke matrix ‚Äî anomalies are findings, not failures). Fix: catch the error but check whether the output JSON file was written; treat the run as OK if it was, and surface stderr/stdout tail only when the output file is absent (e.g. BudgetUnsatisfiable before any runs land).

## Lessons captured

- The swarm runner exits 1 on any anomaly. Matrix drivers must distinguish "run errored before producing output" from "run produced output with findings"; key heuristic is `fs.existsSync(outFile)`.
- The state-cycle detector's snapshot scope is armor+structure+heat. For positional progress (units moving toward each other across an empty map), it produces a steady stream of false positives. Detector-improvement candidate, not a combat-engine defect.
- `BudgetUnsatisfiableError` at high BV / low unit counts is a force-generator + catalog limitation. The matrix needs to widen `unitCount` (e.g. 3-4 units at 10kBV) to exercise the high-BV cell.
- The 50-turn default turnLimit is too short for the r20 + default-AI cell. r20 with regular default AI produces 100% draws. Probably want `turnLimit: 75` or `100` for the next deepening sweep ‚Äî but not blocking Phase 2.
